# Сайт-визитка

Мой личный сайт-портфолио. Написан на Go, без фреймворков, всё в одном бинарнике.

Что тут есть:
- обычное портфолио (обо мне, навыки, проекты, образование)
- блог, что-то вроде мини телеграм-канала — посты с фото, реакции от гостей
- страничка с музыкой — что сейчас играю в Spotify и топ за месяц
- админка, чтобы всё это редактировать без залезания в код

## Стек

- Go (net/http из коробки, роутинг через Go 1.22+)
- SQLite (modernc.org/sqlite, без cgo, так что билдится проще)
- фронт — обычный HTML/CSS/JS без сборки, всё вшито в бинарник через embed
- Telegram бот для уведомлений о новых сообщениях

## Запуск

```bash
cp .env.example .env
# заполни ADMIN_PASSWORD хотя бы

go run .
```

Сайт на http://localhost:8080, админка на /admin.

Собрать бинарник:
```bash
go build -o cvserver .
./cvserver
```

Или через докер:
```bash
docker build -t cv-site .
docker run -p 8080:8080 --env-file .env -v cv-data:/data cv-site
```

## Переменные окружения

Смотри `.env.example`, там всё с комментариями. Коротко:

- `PORT` — порт, по умолчанию 8080
- `DB_PATH`, `UPLOADS_DIR` — куда класть базу и файлы
- `ADMIN_PASSWORD` — пароль в админку, обязательно смени дефолтный
- `TELEGRAM_BOT_TOKEN` / `TELEGRAM_CHAT_ID` — если хочешь уведомления с формы обратной связи
- `SPOTIFY_CLIENT_ID` / `SPOTIFY_CLIENT_SECRET` / `SPOTIFY_REDIRECT_URI` — для блока музыки, опционально

Spotify подключается через админку (кнопка "Подключить"), не через .env напрямую — нужен только client id/secret оттуда, а сам аккаунт коннектится через OAuth.

## Структура

```
main.go              — точка входа, конфиг, роутинг
spotify_poll.go       — фоновая штука, раз в 45 сек проверяет что играет
internal/store/       — вся работа с sqlite (профиль, блог, спотифай токены и т.д.)
internal/api/         — http хендлеры
internal/notify/      — телега бот
internal/spotify/     — клиент для spotify api
web/                  — фронт (index.html, admin.html, assets/)
```

## Заметки

- база — обычный sqlite файл, при первом запуске сама создастся и заполнится моими реальными данными (профиль, проекты и т.д.)
- фото/резюме грузятся через админку, лежат в `data/uploads`
- если что-то не работает — гляни логи, там обычно понятно что не так
