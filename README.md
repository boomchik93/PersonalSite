# Личный сайт-визитка

Мой персональный сайт: портфолио + мини-блог + пара живых интеграций (Spotify, кинотека). Изначально хотел просто выложить резюме, но по ходу решил сделать так, чтобы всё можно было редактировать без деплоя — так появилась админка и база данных. Заодно потренировался писать HTTP-сервер на голом `net/http` без фреймворков.

Всё живёт в одном бинарнике: фронтенд вшит через `embed`, база — SQLite-файл рядом. Скопировал бинарник + папку `data` на сервер и всё, работает.

## Что умеет

- **Портфолио** — обо мне, навыки (сгруппированы), проекты, образование, интересы. Всё тянется одним запросом `GET /api/site`.
- **Резюме** — отдельная страница `/cv`. Если открыть сайт на поддомене `cv.*` (например `cv.semenovm.ru`), резюме показывается сразу на корне.
- **Блог** — что-то вроде личного телеграм-канала: посты с несколькими фото, гости могут ставить реакции. Фото можно переупорядочивать в админке.
- **Кинотека** — `/films`, список просмотренного (фильмы/сериалы). Фильтрация на клиенте, отдаётся всё разом.
- **Spotify** — блок «сейчас играет» + топ треков/артистов. Токены обновляются фоновым поллером.
- **Форма обратной связи** — сообщения падают в базу и, если настроен бот, прилетают уведомлением в Telegram.
- **Админка** (`/admin`) — редактирование всего перечисленного, вход по паролю, сессия на JWT в куке.

## Стек и решения

- **Go 1.25**, стандартный `net/http`. Роутинг — на новом синтаксисе `mux` из Go 1.22 (`GET /api/...`, path-параметры `{id}`), без сторонних роутеров.
- **SQLite** через `modernc.org/sqlite` — чистый Go, без cgo. Из-за этого билд статический (`CGO_ENABLED=0`) и Docker-образ получается крохотный.
- **Фронтенд** — обычные HTML/CSS/JS без сборщика. Вшивается в бинарник директивой `//go:embed all:web`, так что раздавать отдельно нечего.
- **Аутентификация** — JWT собственной сборки (пакет `internal/auth`), подписывается секретом из `JWT_SECRET`.
- **Шифрование** — чувствительные поля (Spotify-токены, email/телефон/телеграм из профиля) лежат в базе зашифрованными AES-256-GCM (`internal/crypto`). Ключ — из `ENCRYPTION_KEY`, без него сервер не стартует.
- **Загрузки** — фото и резюме грузятся в `data/uploads` и раздаются с диска (не вшиты, потому что меняются в рантайме).

## Структура

Раскладка по стандартному layout Go: точка входа в `cmd/`, вся логика — в `internal/`, наружу ничего не торчит.

```
cmd/cvserver/
  main.go            — конфиг, роутинг, graceful shutdown, монтирование статики
  spotify_poll.go    — фоновый поллер, обновляет now-playing и топы
  web/               — фронтенд (index / cv / films / admin + assets), вшивается через embed
internal/
  api/               — HTTP-хендлеры (публичные + админские)
  store/             — весь SQLite: профиль, блог, кинотека, spotify-токены, сообщения
  auth/              — JWT для сессий админки
  crypto/            — AES-256-GCM для полей в базе
  notify/            — Telegram-бот
  spotify/           — клиент Spotify API + OAuth
```

## Запуск

Нужен Go 1.25+.

```bash
cp .env.example .env
# как минимум заполни ADMIN_PASSWORD и ENCRYPTION_KEY
# ключ можно сгенерить: openssl rand -base64 32

go run ./cmd/cvserver
```

Сайт поднимется на http://localhost:8080, админка — на `/admin`.

> При первом старте, если `ENCRYPTION_KEY` не задан, сервер напечатает свежесгенерированный ключ и завершится — просто скопируй его в `.env` и запусти снова. Ключ нельзя менять после того, как в базе появились зашифрованные данные, иначе они станут нечитаемыми.

Собрать бинарник:

```bash
go build -o cvserver ./cmd/cvserver
./cvserver
```

Через Docker (образ на `alpine`, сборка статическая):

```bash
docker build -t cv-site .
docker run -p 8080:8080 --env-file .env -v cv-data:/data cv-site
```

## Переменные окружения

Полный список с комментариями — в `.env.example`. Ключевое:

| Переменная | Обязательна | Назначение |
|---|---|---|
| `ENCRYPTION_KEY` | да | AES-256 ключ (base64, 32 байта) для шифрования полей в базе. Без него старта нет. |
| `ADMIN_PASSWORD` | да | Пароль в админку. Дефолт `admin` — сервер ругнётся, поменяй. |
| `JWT_SECRET` | желательно | Секрет подписи сессий (base64, 32+ байта). Если не задан — генерится случайный на каждый старт, и сессии не переживают рестарт. |
| `PORT` | нет | Порт, по умолчанию `8080`. |
| `DB_PATH` / `UPLOADS_DIR` | нет | Куда класть базу и загрузки. Папки создаются сами. |
| `TELEGRAM_BOT_TOKEN` / `TELEGRAM_CHAT_ID` | нет | Уведомления о сообщениях с формы. Без них форма всё равно пишет в базу. |
| `SPOTIFY_CLIENT_ID` / `SPOTIFY_CLIENT_SECRET` / `SPOTIFY_REDIRECT_URI` | нет | Для блока с музыкой. |

Сам аккаунт Spotify подключается один раз через админку (вкладка Spotify, OAuth) — в `.env` кладётся только client id/secret и redirect URI.

## Заметки

- База — обычный SQLite-файл, создаётся и наполняется автоматически при первом запуске.
- Всё содержимое (профиль, проекты, посты, фильмы) правится через админку, лезть в код не нужно.
- Сервер логирует каждый запрос и корректно завершается по SIGINT/SIGTERM (успевает закрыть базу и остановить поллер).
- Если что-то отвалилось — первым делом смотрю логи, там обычно сразу видно причину.
